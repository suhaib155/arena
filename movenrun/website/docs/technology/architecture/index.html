Architecture overview

MovenRun is a TypeScript monorepo: one app, one shared contract of types, one verification backend, one experimental chain workspace, and one static website — connected only where the product has earned the connection.

Current vs. future connections

  • Mobile ↔ backend: not connected. The app is fully on-device today; route submission to the pipeline is a Phase 3 milestone.
  • Backend ↔ chain: read-only. A no-signer client can read the Base Sepolia deployment; the oracle-signing path exists in code and is exercised by tests, not by users.
  • Mobile ↔ chain: display-only. One read-only status screen and an educational deed showroom. No wallet, no writes.
  • Website ↔ everything: informational. The site and docs are static and make no API calls.

Architectural principles

  • The seam pattern. The app consumes data through service seams so on-device simulations can later be swapped for verified backend data without UI rewrites.
  • Characterize before redesign. The deployed V1 contracts are treated as an immutable experiment: their defects are being documented (PR #44) before the isolated V2 (PR #45) replaces them — never patched in place.
  • Static by default. Web surfaces are dependency-free static files; no frameworks, no build pipeline.
