Contracts: architecture & status

One experimental V1 suite on a test network, one honest catalogue of its defects, and one isolated V2 redesign — each at a different, clearly-labeled stage.

The contract architecture

The suite models the future territory economy as eight cooperating contracts:

Contract roles (V1 names; V2 keeps the same shape)
ContractStandardRole
Move tokenERC-20The in-game token concept; minting gated by oracle-verified movement
GPS oracle—Verifies backend attestations for movement-derived actions
Zone deed (ZoneNFT)ERC-721Territory records; token ID = zone cell ID; dormancy/reclaim
GearERC-1155Equipment with bounded stat multipliers
Zone challenge—Time-boxed territory challenge/defense battles
Season controller—90-day seasons, emission valve, season-end events
Vault—Staking and treasury accounting
DAO / Governor—Governance (V1: custom tiers; V2: standard snapshot governance + timelock)

V1 — experimental testnet Experimental testnet

The V1 suite is deployed on Base Sepolia only, with the deployment record and address registry on main. It is experimental infrastructure: not mainnet, not production-ready, not audited, and not a public financial product. Nothing in the consumer app writes to it.

A formal characterization effort — PR #44, Under review — proves the deployed V1's behavior with a passing test suite and catalogues 16 known discrepancies (2 critical, several high) across the challenge lifecycle, season emissions, zone/gear state, and vault/DAO accounting. At a high level: challenges can be disrupted or replayed, a "burn" that doesn't reduce supply, state that leaks across deed reclaims, and governance that isn't snapshot-safe. The docs deliberately stop at this altitude — the full report exists for reviewers, and these pages do not publish exploit walkthroughs. Do not interact with the V1 testnet contracts expecting sound behavior.

V2 — under review Under review

PR #45 adds an isolated V2 suite: source and tests only; not merged and not deployed. V1 remains untouched and remains the only deployed suite. The V2 design's intended improvements, at a high level:

  • An explicit challenge lifecycle (one active challenge per zone, idempotent resolution, no overwrites) with challenger wins that settle without defender cooperation.
  • EIP-712 typed signatures everywhere — domain-bound, deadline-bound, nonce-bound; V1-style signatures are rejected.
  • Season pauses that actually stop minting on every path; a season-end burn that genuinely reduces supply, exactly once.
  • Live-ownership gear multipliers; checkpointed vault accounting that survives under-funding.
  • Standard snapshot governance (OpenZeppelin Governor + timelock) replacing the custom live-balance DAO.

V2 is not audited (its 146 passing tests are not an independent audit), not deployed, and no migration from V1 has happened. If PR #45's state changes, this page changes with it.

Standing rules

  • Deployed contracts are audited before they are changed; defects are characterized, not silently patched.
  • No deployment happens inside feature PRs; deployment records are the only proof of deployment the docs accept.
  • Independent audit and legal review gate anything beyond testnet — see Release gates.
