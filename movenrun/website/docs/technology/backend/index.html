Movement verification

Ownership is only as trustworthy as the movement behind it. MovenRun's verification layer turns a recorded route into a proof the whole economy can rely on.

Why it matters

If territory is a real asset, the network can't reward movement it can't trust. Verification is what keeps the map fair, the economy healthy, and ownership meaningful — it's the difference between a game of effort and a game of exploits.

The proof-of-movement pipeline

  1. Validate. The route is checked for physical plausibility — realistic speed, sane accuracy, continuous motion, sane duration.
  2. Map. The path is resolved to the ordered set of zones it crossed (see Territory computation).
  3. Reduce. The session is distilled into a compact, deterministic proof — a fingerprint of the movement, not a stored trail.
  4. Attest. The oracle signs the proof so onchain contracts can settle capture and rewards without trusting the client.
  5. Guard. Duplicate and replayed proofs are rejected, so the same movement can't be claimed twice.

The oracle

The oracle is the bridge between the physical world and the chain: it attests, with a network-held signing key, that a route passed verification, and its authority is scoped and rotatable. Onchain contracts require that attestation before acting on movement — so real-world activity can drive ownership without the chain ever trusting an app.

Anti-spoofing and integrity

  • Plausibility checks catch vehicles, teleports, and synthetic traces.
  • Deterministic proofs plus deduplication stop replays and double-claims.
  • Layered defense — verification is designed to keep hardening as the network grows, because integrity is the foundation the whole economy stands on.
