Development history

MovenRun has been built in public, in 46 pull requests. Here is that history as milestones — with open work clearly separated from shipped work.

Audited from the live GitHub record on 12 July 2026: 46 PRs total — 42 merged, 2 open, 2 closed unmerged.

Milestone 1 — Foundation (May–June 2026)

The monorepo scaffold — contracts, backend, shared types, and mobile — landed in PR #1. A first mobile MVP (movement quests, XP, streaks, onboarding) followed in PRs #2–4, with device-testing docs and the EAS Android build pipeline in PRs #6–8. Two early chores were closed unmerged (#5, later #39) — superseded approaches, kept honest in the record.

Milestone 2 — Direction and on-chain reconciliation (June 2026)

PR #9 realigned the roadmap around the territory economy. PR #10 reconciled the already-deployed Base Sepolia V1 contracts onto main: deployed source, deployment record, and the shared address registry — no redeploy. The cinematic website (PR #11) and the app's Daylight Cartography design system (PR #12) established the brand.

Milestone 3 — The movement loop (June 2026)

The heart of the product landed as a chain of mobile PRs: GPS sessions (#13), mock H3 territory capture (#14), the defend/fortify loop (#15), and clubs with a local leaderboard preview (#16). Everything explicitly on-device and preview-labeled.

Milestone 4 — Verification previews and trust UX (June 2026)

Read-only chain infrastructure (#17) and the in-app testnet status screen (#18) made the contract foundation visible without a wallet. Route Trust previews (#19), review history (#20), the signal passport (#21), and the shareable proof card (#22) built the verification story locally, with privacy-safe summaries only.

Milestone 5 — Depth and the city layer (June–July 2026)

Onboarding questline (#23), territory map (#24), collections (#25), cinematic opening (#26), zone command center (#27), territory alerts (#28), weekly recap (#29), season objectives (#30) — then the city layer: districts (#31), rival ghosts (#32), city war board (#33), sponsor zones (#34), event zones (#35), club territory (#36), crew missions (#37), and district mastery (#38). All local previews with fictional data, labeled as such in-app.

Milestone 6 — Backend hardening (July 2026)

Three backend PRs made the verification pipeline real infrastructure: oracle-signature alignment with the deployed contracts (#40), route-lifecycle persistence with server-side deduplication (#41), and wallet-signature auth, rate limits, security headers, and a CORS allowlist (#42). The deed preview showroom (#43) closed the loop on educational ownership UX.

Milestone 7 — Public face (July 2026)

PR #46 repositioned the website health-first, added responsive navigation and a professional footer, and removed speculative language — the live site today.

Incoming work — open, under review, not shipped

The two pull requests below are open. Nothing in them is merged, deployed, or part of the current product. They are documented here so the docs never present branch work as shipped.

  • Under review PR #44 — audit(contracts): characterize deployed v1 lifecycle discrepancies. Characterization and documentation only: 18 tests proving the deployed V1's known defects (16 catalogued issues), plus a written report. No Solidity changes, no deployment.
  • Under review PR #45 — fix(contracts): add isolated v2 territory economy contracts. A parallel V2 contract suite — source and tests only (146 passing). V1 untouched, nothing deployed, no mobile/backend/website integration.

Closed without merging

  • PR #5 — Expo phone-testing chore; superseded by the documentation approach in PR #6.
  • PR #39 — Vercel web deployment for the mobile app; abandoned in favor of keeping the website as the only web surface.
