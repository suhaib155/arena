Current product status

One page, one truth. Every feature below is placed in exactly one status — using the same vocabulary as its badge everywhere else in these docs.

Last reviewed: 12 July 2026 · audited against main @ cf6ca3f and the full pull-request record.

Status vocabulary

What each status badge means
BadgeMeaning
ImplementedMerged into main and part of the relevant current build or service
Local previewWorks on-device with deterministic or fictional data; no live backend, multiplayer, payments, rewards, or chain behavior
InfrastructureBackend/deployment/security capability exists, but is not connected to the public mobile experience
Experimental testnetDeployed or testable only on a test network; not production, not mainnet, not a financial product
Under reviewExists in an open pull request; not merged
PlannedApproved direction, not implemented
ConditionalOnly considered after explicit release gates (density, verification, audit, legal review, demand)
HistoricalEarlier design or implementation kept for context; not the current promise
