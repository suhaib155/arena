Glossary & references

The terms these docs use, defined once — followed by the sources they are built from.

Glossary

Move session
A recorded period of walking, running, or cycling: route, distance, duration, pace.
Zone
One hexagonal cell of the map grid — the atom of territory. A digital game concept, never real land.
Territory
The set of zones a player has captured and maintains.
Capture
Claiming a zone by finishing a session whose route touched it. Currently an on-device simulation.
Defense
A zone's resistance to fading (and, in the future model, to challenges). Decays with neglect; refreshed by returning.
Fortify
An explicit action that strengthens a zone you hold.
District
A named group of zones at city scale, with its own mastery progression. Currently fictional groupings.
Club
A team players join for shared motivation and (future) shared territory. Currently a local preview with four fictional clubs.
Crew mission
A weekly team-style goal generated from territory activity. Local preview.
City war
A season-long, city-scale competition between clubs. Currently a fictional preview board.
Zone Deed
A proposed on-chain territory record (ERC-721) for committed players. Local showroom + experimental testnet only; never required; not land.
Movement points / Locked MOVE
The in-app progress counter shown on summaries. In the current build, a display preview derived from XP, labeled “not a payout”. Not a token; nothing is stored or redeemable.
Liquid MOVE
A historical design term for a capped, transferable reward concept for deed holders — strictly gated, never implemented, and not promised.
Testnet
A public blockchain for experiments, with valueless tokens. MovenRun's V1 contracts are on Base Sepolia.
Mainnet
A production blockchain carrying real value. MovenRun has no mainnet deployment.
Oracle
The backend acting as a signed witness that a route passed verification, so contracts can require proof of movement.
H3
The open-source hexagonal geospatial grid system used to define zones.
Route hash
A deterministic fingerprint (SHA-256) of a route used for deduplication and replay prevention.
V1
The first contract suite, deployed to Base Sepolia. Experimental; 16 documented discrepancies.
V2
The redesigned contract suite fixing V1's defects. Under review: source and tests only, not merged, not deployed.
